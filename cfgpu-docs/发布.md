# 发布与部署

本文档描述 cfgpu Consumer 系统的发布方式。

---

## 架构概览

```
[RocketMQ: $AGENT_TASKS] ──► [Consumer 1] ─┐
                          ──► [Consumer 2] ─┼──► [PostgreSQL :5432]  (网络)
                          ──► [Consumer N] ─┘
                                            └──► [DEER_FLOW_HOME]    (共享文件系统)
```

- **PostgreSQL**：唯一共享数据库实例，存储 LangGraph checkpoints、deerflow ORM 表及 Consumer 自定义表
- **DEER_FLOW_HOME**：共享文件系统，存储 config.yaml、agent 定义、skills、channels 等
- **Consumer**：无状态 worker，可横向扩容，全部连接同一 PostgreSQL 和同一 DEER_FLOW_HOME
- **RocketMQ**：负责请求分发，Consumer 动态增减后自动负载均衡
- **不使用** nginx、deerflow FastAPI gateway、前端

---

## 共享依赖说明

Consumer scaling 依赖两类共享资源，性质不同：

| 共享依赖 | 类型 | 共享方式 |
|---------|------|---------|
| PostgreSQL | 网络连接 | 所有 consumer 使用相同的 `DATABASE_URL`，连接同一 postgres 进程 |
| DEER_FLOW_HOME | 文件系统 | 所有 consumer 挂载同一目录（本地 bind mount 或云端 NFS/EFS） |

### DEER_FLOW_HOME 目录内容

| 路径 | 内容 | 是否需要共享 |
|------|------|------------|
| `config.yaml` | 应用配置 | 是，所有 consumer 必须用同一套配置 |
| `.deer-flow/agents/` | agent 定义（如 `director/config.yaml`） | 是 |
| `.deer-flow/skills/` | skill 定义 | 是 |
| `.deer-flow/channels/` | channel 配置 | 是 |
| `.deer-flow/data/` | SQLite 数据文件 | postgres 模式下不使用 |

### .env 的处理

`.env` 包含密钥（API keys、DB URL、RocketMQ endpoint），**不应作为共享文件挂载**，而是通过容器编排注入相同的环境变量，每个 consumer container 独立持有。

## 本地开发

### 1. 启动 PostgreSQL

```bash
docker run -d \
  --name deerflow-postgres \
  -e POSTGRES_USER=deerflow \
  -e POSTGRES_PASSWORD=deerflow \
  -e POSTGRES_DB=deerflow \
  -p 5432:5432 \
  -v deerflow-pgdata:/var/lib/postgresql/data \
  postgres:16
```

`-v deerflow-pgdata` 挂载 named volume，容器重启数据不丢失。

### 2. 配置环境变量

`.env`：
```
DATABASE_URL=postgresql://deerflow:deerflow@localhost:5432/deerflow
DEER_FLOW_HOME=/path/to/deer-flow-home
```

### 3. 安装 postgres 依赖

```bash
cd backend
uv sync --extra postgres
```

### 4. 修改 config.yaml

```yaml
database:
  backend: postgres
  postgres_url: $DATABASE_URL
  pool_size: 10        # 每个 Consumer 实例的连接池大小

run_events:
  backend: db
  max_trace_content: 10240
  track_token_usage: true
```

> **pool_size 说明**：N 个 Consumer 实例 × pool_size = PostgreSQL 总连接数，需在 PostgreSQL `max_connections`（默认 100）范围内。

### 5. 建表

Consumer 启动时自动建表，无需手动执行 migration：
- deerflow ORM 表：`Base.metadata.create_all` 在 `init_engine` 时执行
- LangGraph checkpoint 表：`AsyncPostgresSaver.setup()` 在 Consumer 启动时执行
- Consumer 自定义表（`consumer_instances`、`thread_run_state`、`thread_inject_queue`、`thread_cancel_signals`、`processed_messages`）：`CREATE TABLE IF NOT EXISTS`，幂等安全，多实例并发启动不冲突

## 生产部署

### docker-compose.yml

```yaml
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_USER: deerflow
      POSTGRES_PASSWORD: deerflow
      POSTGRES_DB: deerflow
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  consumer:
    build: ./backend
    command: python -m consumer
    env_file: .env                          # 注入相同的环境变量（含 DATABASE_URL、API keys 等）
    volumes:
      - /shared/deer-flow-home:/app/deer-flow-home   # 挂载共享文件系统
    environment:
      DEER_FLOW_HOME: /app/deer-flow-home
    depends_on:
      - postgres

volumes:
  pgdata:
```

共享文件系统选择：
- **单机部署**：主机目录 bind mount（`- ./deer-flow-home:/app/deer-flow-home`）
- **多节点部署**：NFS、AWS EFS、阿里云 NAS 等网络文件系统

### 扩容

```bash
# 启动 5 个 Consumer 实例
docker compose up --scale consumer=5

# 缩容
docker compose up --scale consumer=2
```

Consumer 实例增减后，RocketMQ 消费组自动重新分配队列。每个实例启动时向 `consumer_instances` 表注册，下线时更新 status；PostgreSQL 的 `thread_run_state` 保证同一 thread 同一时刻只有一个 Consumer 持有执行权（`SELECT FOR UPDATE` 原子 claim）。

---

## 注意事项

- **不能多个 PostgreSQL 实例共享同一数据目录**：多进程写同一 pg data 目录会导致数据损坏。扩容只扩 Consumer，postgres 始终一个实例。
- **SQLite 不适用于多 Consumer**：SQLite WAL 模式仅适合同进程并发，多进程/多节点必须使用 `backend: postgres`。
- **连接数上限**：调整 `pool_size` 时确保 N × pool_size < PostgreSQL `max_connections`。如需更多连接，可在 pg 侧调整 `max_connections` 或引入 PgBouncer 连接池代理。
- **config.yaml 变更**：修改 `DEER_FLOW_HOME` 下的 `config.yaml` 后，所有 consumer 实例自动感知（deerflow 按文件 mtime 热重载），无需重启。
